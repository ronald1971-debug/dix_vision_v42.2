## Core Philosophy

**Vision Statement:** DIX VISION is a Market Cognitive Operating System (MCOS) that treats the market as a living cognitive environment. The system continuously observes, models, learns, evolves, and understands that environment. Trading is one expression of the system's understanding, not the primary goal.

**Primary Objectives:**
- Develop market intelligence, trader intelligence, system intelligence, and self-understanding
- Become the ultimate trader through continuous cognitive evolution
- Maintain operator authority and constitutional governance
- Ensure deterministic, auditable, and safe operations

**Core Principles:**
- Event-sourced design (no hidden state)
- Strict domain separation (INDIRA owns markets, DYON owns system)
- BeliefState as single source of truth
- Operator has final authority
- Continuous learning and evolution
- Governance as constitutional authority

### 2. Risk Controls

**Features:**

- Position size limits
- Drawdown limits
- Volatility limits
- Correlation limits

### 3. Mode State Machine

**Canonical Mode Model:**
```
SAFE → PAPER → SHADOW → CANARY → LIVE → AUTO → LOCKED
```

**Mode Behaviors:**
- SAFE: Analysis only, risk-only
- PAPER: Simulated execution
- SHADOW: Live signal, no execution
- CANARY: Small capital live
- LIVE: Full governed trading
- AUTO: Autonomous execution under governance constraints
- LOCKED: Frozen system

**Strict Rules:**
- Single source of mode truth (GovernanceEngine.state.mode)
- OPERATOR has ultimate authority to initiate mode transitions
- Governance validates and executes operator's mode transition decisions
- Dashboard is only control surface (emits events, doesn't execute)
- Mode effects are global invariants

### 4. Authority Boundaries

**Strict Domain Separation:**
- INDIRA → market execution only
- DYON → system infrastructure + hazard detection only
- Governance → all decision translation
- SYSTEM_HAZARD → emergency signal channel only

**No component can directly bypass Governance**

- Rollback mechanisms
- Safe failure modes
